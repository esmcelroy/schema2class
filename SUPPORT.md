# Support

Use the channel that matches the request:

- Bug reports: open a GitHub issue with a minimal schema, command or Gradle
  configuration, expected output, and actual output.
- Feature requests: open a GitHub issue describing the schema feature, target
  generated Kotlin, and any compatibility constraints.
- Documentation problems: open a GitHub issue or pull request with the affected
  page and suggested correction.
- Security vulnerabilities: follow [SECURITY.md](SECURITY.md) instead of using a
  public issue.

For general usage questions, prefer GitHub Discussions if enabled. Otherwise,
open a question-style issue and include the schema format, annotation mode, and
schema2class version.
